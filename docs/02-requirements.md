# 02. 요구사항

## 기능 요구사항 (FR)

### FR-1. 데이터 수집
- **일봉/분봉 시세** 수집 및 로컬 저장 (KOSPI/KOSDAQ 전 종목)
- **실시간 시세** 구독 (매매 대상 종목 한정)
- **재무제표** 분기별 수집 (DART)
- **공시** 실시간 수신
- **뉴스** 수집 (네이버 뉴스/RSS)
- **수급 데이터** 외국인·기관·개인 (일별)

### FR-2. 신호 생성
- 전략별 독립된 신호 생성 모듈
- 신호 = `{종목코드, 방향(매수/매도), 비중, 근거, 생성시각}`
- 다전략 실행 시 **신호 병합 규칙** 필요 (중복·상충 처리)

### FR-3. 주문 집행
- 시장가 / 지정가 / IOC / FOK 지원 (증권사 API 한도 내)
- **주문 멱등성 보장**: 재시도 시 중복 주문 방지
- 주문 상태 추적: 접수·체결·부분체결·취소
- 장 시간 외 주문은 큐잉 후 장 시작 시 실행

### FR-4. 포지션 관리
- 현재 보유 수량·평단·미실현 PnL 실시간 집계
- 증권사와의 잔고 정합성 검증 (장 시작 전 / 주기적)
- 배당·액면분할·감자 등 코퍼레이트 액션 반영

### FR-5. 백테스팅
- 과거 데이터 기반 전략 성과 시뮬레이션
- 슬리피지·수수료·거래세 반영
- 워크포워드 분석 지원
- 결과 리포트 자동 생성

### FR-6. 모니터링 / 알림
- 대시보드: 현재 포지션, 일별 PnL, 최근 주문, 전략별 성과
- 알림: 주문 체결, 손절 발동, 이상 상황(텔레그램/슬랙/이메일)

### FR-7. 감사 로그
- 모든 신호·주문·체결·리스크 이벤트 **append-only** 기록
- 사후 분석·재현·책임 추적 가능해야 함

## 비기능 요구사항 (NFR)

### NFR-1. 성능
- 시그널 생성 → 주문 전송: **1초 이내** (인트라데이 전략 기준)
- 백테스트: 1년치 일봉 × 100종목 < 10초 (vectorized 기준)
- **HFT가 아니므로 밀리초 지연은 목표 아님**

### NFR-2. 가용성
- **장중(09:00~15:30) 99% 가용성** 목표
- 장 외 시간 다운은 허용 (배치 작업만 수행)
- 치명적 장애 시 **안전 정지**가 우선 (무리한 복구 시도 금지)

### NFR-3. 신뢰성
- 네트워크·API 단절 시 재연결 로직 (exponential backoff)
- 주문 중복·누락 방지 (멱등키)
- 시세 데이터 결손 탐지 및 알림

### NFR-4. 관측 가능성
- 구조화 로그 (JSON) + 로그 레벨
- 주요 메트릭: 주문수, 체결률, API 응답시간, PnL
- 에러 추적: 스택트레이스 + 컨텍스트

### NFR-5. 보안
- API 키는 **절대 저장소에 커밋하지 않음**
- 환경변수 또는 OS 키체인 사용
- 계좌번호 등 민감정보는 로그에서 마스킹
- 자세한 내용: [12-security.md](./12-security.md)

### NFR-6. 테스트 가능성
- 전략 로직은 순수 함수 지향 (결정적)
- 주문·시세 모듈은 모의(mock) 구현 제공
- 모의투자 계좌로 **E2E 리허설** 가능

### NFR-7. 유지보수성
- 파이썬 3.11+, 타입 힌트 필수
- `pydantic` 모델로 경계(boundary) 데이터 검증
- 전략 추가 시 기존 코드 수정 없이 플러그인 형태로 등록

## 제외 요구사항 (Explicitly Not Required)

- 다중 사용자 지원
- 웹 기반 계정 관리
- 모바일 앱
- 실시간 협업 기능
