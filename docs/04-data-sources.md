# 04. 데이터 소스

## 개요

자동매매 시스템의 질은 데이터의 질로 결정된다. 한국 시장은 공개 데이터가 비교적 풍부하지만 **출처별로 지연·정확도·이용약관**이 다르므로 용도에 맞게 조합한다.

## 시세 데이터

| 출처 | 종류 | 지연 | 비용 | 비고 |
|---|---|---|---|---|
| **KIS Open API (WebSocket)** | 실시간 틱/호가 | 실시간 | 무료 | 계좌 필요. 1순위 |
| **KIS Open API (REST)** | 과거 분봉·일봉 | - | 무료 | 조회 한도 있음 |
| **pykrx** | 일봉, 수급, 공매도 | T+1 | 무료 | KRX 공식 데이터 스크래핑 |
| **FinanceDataReader** | 일봉 (한·미) | T+1 | 무료 | 간편. 백테스트 1차 소스 |
| **네이버 금융** | 일봉, 분봉 | T+1 | 비공식 | 약관·안정성 주의 |

**선택**:
- 실시간: KIS WebSocket
- 과거 일봉 적재: `pykrx` + `FinanceDataReader` 교차 검증
- 과거 분봉: KIS REST (종목·기간 한정 적재)

## 뉴스 데이터

| 출처 | 장점 | 단점 | 도입 시점 |
|---|---|---|---|
| **네이버 뉴스 검색 API** | 공식, 간편, 키 발급 즉시 사용 | 일 호출 한도, 순위 기반이라 최신성 제한 | **Phase 1** |
| **언론사 RSS** (한경·매경·연합) | 최신, 무료, 키 불필요 | 출처별 포맷 상이, 종목 매칭 별도 | **Phase 1** |
| **KIS 종목뉴스 API** | 보유 계좌로 직접 호출, 종목 매핑 사전 처리됨 | 종목 단위 호출 (브로드캐스트 X), **KIS 계정 필요** | Phase 4 |
| **Google 뉴스 RSS** | 광범위 | 약관 주의, 안정성 낮음 | 검토만 |
| **유료 API (Aicel, BigKinds 등)** | 정제된 데이터 | 비용 | Phase 11+ |

**선택**:
- **Phase 1 MVP**: 네이버 뉴스 검색 API + 언론사 RSS — KIS 계정 없이 즉시 시작 가능
- **Phase 4 추가**: KIS 종목뉴스 API (KIS 계정 발급 이후, 동일 수집 인터페이스에 어댑터로 연결)
- 뉴스-종목 매핑은 [07-analysis-engines.md](./07-analysis-engines.md) 참조

```
  kis/
  dart/

  ohlcv_daily

```

## 데이터 품질 체크

- **결손 탐지**: 거래일인데 데이터 없음 → 알림
- **이상치**: 가격 변동률이 상/하한가 초과 → 플래그
- **생존편향(survivorship bias) 방지**: 상장폐지 종목도 백테스트에 포함
- **리스테이트먼트**: 수정된 재무제표가 있으므로 `as-of` 날짜 관리 필요 (MVP에서는 최신값 사용)

## 약관 및 레이트 리밋

| API | 제한 |
|---|---|
| KIS REST | 초당 20건 내외 (API별 상이) |
| KIS WebSocket | 동시 구독 종목 수 제한 |
| DART | 일 10,000건 |
| 네이버 검색 | 일 25,000건 (기본) |

**설계 반영**:
- Rate limiter는 `asyncio.Semaphore` 또는 `aiolimiter`
- 초과 시 큐잉 + 지수 백오프
- 장 시작 전에는 집중 호출 피함 (증권사도 바쁨)
