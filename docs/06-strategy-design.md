# 06. 전략 설계

## 공통 인터페이스

모든 전략은 동일한 계약을 구현하여 Strategy Engine에서 동일하게 실행된다.

```python
class Strategy(Protocol):
    name: str
    universe: list[str]          # 대상 종목 목록
    timeframe: Timeframe         # daily / minute / event-driven

    def generate_signals(
        self,
        data: MarketData,
        context: StrategyContext,  # 현 포지션, 현금, 최근 신호 등
    ) -> list[Signal]: ...
```

신호(`Signal`)는 **결정 + 근거**를 함께 기록한다:

```python
Signal(
    symbol="005930",
    side="BUY" | "SELL",
    target_weight=0.05,          # 또는 수량
    confidence=0.7,
    rationale="RSI oversold + 20일선 지지",
    strategy="mean_reversion_v1",
    generated_at=datetime.now(),
)
```

**원칙**: 신호 생성과 주문 실행은 분리. 신호는 전략의 뜻이고, 실제 주문은 Risk Engine이 판단한다.

## 스타일 1. 단기 (인트라데이 / 스윙)

### 예시 전략
- **이동평균 크로스오버**: 5일선 vs 20일선
- **RSI 평균회귀**: RSI < 30 매수 후보, RSI > 70 매도 후보
- **볼린저 돌파**: 밴드 상단 돌파 = 추세, 하단 이탈 = 과매도
- **VWAP 기반 인트라데이**: 분봉에서 VWAP 대비 편차

### 주의
- **수수료·거래세(0.18%)·슬리피지**가 이익을 갉아먹음 → 1회 목표수익 최소 1% 이상
- 한국장은 **동시호가(08:30~09:00, 15:20~15:30)** 특성상 시가/종가 변동성 큼
- 상/하한가(±30%) 근처는 유동성 급변 주의
- 인트라데이는 포지션을 **장 마감 전 청산**하는 규칙이 기본 (오버나잇 갭 리스크 회피)

### 데이터 요구사항
- 분봉 시세 (5분/1분)
- 실시간 호가
- 전일 종가·이동평균

## 스타일 2. 중장기 (펀더멘털)

### 예시 전략
- **가치주 스크리너**: PER < 업종 평균, PBR < 1.0, ROE > 10%
- **배당주 포트폴리오**: 시가배당률 상위 + 배당성장
- **실적 서프라이즈**: 컨센서스 대비 실적 초과 종목

### 리밸런싱
- 월 1회 또는 분기 1회
- 리밸런싱 비용(수수료+세금) 고려해 **편입·편출 임계값** 설정

### 주의
- 재무데이터는 **공시 시차**(분기 종료 후 45일 이내) 있음
- `as-of` 날짜 관리로 **룩어헤드 편향(lookahead bias)** 방지
- 상장폐지 종목도 유니버스에 포함해야 **생존편향** 제거

### 데이터 요구사항
- 분기 재무제표 (DART)
- 일별 가격·시총·PER/PBR (pykrx)
- 업종 분류

---

## 스타일 3. 뉴스 / 이벤트 기반

### 예시 전략
- **실적 공시 반응 매매**: 어닝 서프라이즈 → 단기 모멘텀
- **자사주 매입 공시 후 진입**: 역사적으로 양의 효과
- **유상증자 공시 후 매도**: 희석 영향
- **뉴스 감성 점수 기반**: 긍정 뉴스 급증 종목 매수

### 파이프라인
```
공시/뉴스 수신 → 종목 매칭 → 이벤트 분류 → 감성·중요도 스코어 → 신호
```

### 주의
- **뉴스는 이미 반영되었을 수 있음** (정보 비대칭 관리)
- 공시 시각 vs 시장 반응 시각 정확히 기록
- 같은 이벤트가 여러 출처에서 중복 수신됨 → **중복 제거**(제목 해싱/임베딩)
- 악재에 기계적으로 매도하면 **패닉 셀 추격** 위험

### 데이터 요구사항
- DART 실시간 공시
- 뉴스 (네이버/RSS)
- 종목 매칭 사전 (종목명 별칭·모회사 관계)
- 감성 분석 모델 → [07-analysis-engines.md](./07-analysis-engines.md)

## 스타일 4. 퀀트 / 팩터

### 멀티팩터 프레임워크

일반적 팩터 카테고리:
- **Value**: PER, PBR, EV/EBITDA 낮을수록 선호
- **Momentum**: 12-1개월 수익률
- **Quality**: ROE, 부채비율, 이익안정성
- **Low Volatility**: 과거 변동성 낮을수록 선호
- **Size**: 시가총액 (한국은 소형주 프리미엄 논의 활발)

### 팩터 조합
1. 각 팩터별 종목 순위 → z-score 정규화
2. 가중 합산 (동일가중 또는 변동성 역가중)
3. 상위 N종목 선택, 동일가중 또는 변동성 기반 비중

### 리밸런싱
- 월 1회 표준. 전환율(turnover) 모니터링 필수
- 거래비용이 이론 알파를 크게 갉아먹음

### 주의
- **과적합(overfitting)**: 팩터 많을수록 과거에 맞추기 쉬움
- **팩터 크라우딩**: 유명 팩터는 이미 알파가 희석됨
- 한국 시장은 팩터 통계적 유의성이 미국보다 낮은 경우 많음

### 데이터 요구사항
- 과거 가격 + 재무 패널 데이터
- 생존편향 제거된 유니버스

## 다전략 통합

### 자본 배분
- 전략별 **고정 비율**로 자본 할당 (예: 단기 20%, 중장기 40%, 뉴스 20%, 퀀트 20%)
- 전략 성과에 따라 주기적 재조정 (6개월)

### 신호 충돌 처리
- 동일 종목에 상반 신호 → **우선순위 규칙** 또는 **비중 합산 후 방향 결정**
- 기본 정책: **중장기 포지션은 단기 신호로 청산하지 않음**

### 상관관계 관리
- 전략 간 수익률 상관 모니터링. 높으면 다변화 효과 없음
- 모든 전략이 동시에 매수 신호를 내는 구간(= 시장 전체 강세) 주의

## Dev 원칙

- 각 전략은 **독립 모듈**, 디렉토리 한 개
- 전략 파라미터는 YAML/JSON으로 외부화 (`strategies/mean_reversion/config.yaml`)
- 전략 추가는 **플러그인 등록**으로 (`entry_points` 또는 내부 레지스트리)
- 신규 전략은 백테스트 통과 → 모의투자 2주 → 실전 투입 순서 ([14-roadmap.md](./14-roadmap.md))
