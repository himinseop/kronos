# 07. 분석 엔진 — 뉴스 NLP & 기업평가

## 1부. 뉴스·공시 분석

### 파이프라인 전체

```
[수집]  DART 공시 / 네이버 뉴스 / RSS
    ↓
[정규화]  제목·본문·메타 추출, 중복 제거 (해싱 + 근사 중복)
    ↓
[종목 매칭]  본문 → 종목코드 리스트
    ↓
[분류]  이벤트 타입 (실적/증자/M&A/소송 etc.)
    ↓
[감성·중요도 스코어]  점수화
    ↓
[신호 생성]  전략 규칙 적용
```

### 단계별 접근

#### 1단계: 키워드·룰 기반 (MVP)

- 종목명 사전 + 별칭 맵 (예: "삼전" → 삼성전자)
- 이벤트 키워드 사전:
  - 긍정: "영업이익 증가", "호실적", "신기록", "계약 체결"
  - 부정: "영업손실", "감사의견 거절", "상장폐지", "횡령"
- 단순 점수: (긍정 키워드 수) - (부정 키워드 수)

**장점**: 투명, 빠름, 해석 가능
**한계**: 문맥 무시, 어휘 누락

#### 2단계: 학습된 감성 분류기

- **KR-FinBERT** 우선 사용 (한국어 **금융 도메인 특화** 사전학습 모델)
  - 예: `snunlp/KR-FinBert-SC` (HuggingFace) — 한국어 금융 뉴스로 파인튜닝됨
  - 일반 한국어 감성 모델(KoBERT/KoELECTRA)은 "호실적·어닝쇼크·유상증자" 같은 금융 어휘 맥락을 잘 잡지 못해 부적합
- 폴백: KoBERT/KoELECTRA + 금융 데이터로 자체 파인튜닝 (KR-FinBERT 사용 불가 시)
- 입력: 제목 또는 제목+첫 문단
- 출력: positive / negative / neutral + confidence
- **추가 라벨링**: 자체 데이터 100~300건 라벨링 후 도메인 적응(추가 학습) 권장

#### 3단계: LLM 기반 이벤트 추출 (선택)

- Claude/GPT API로 공시 본문 요약 + 이벤트 구조화
- 예시 프롬프트: "이 공시의 종목 영향(긍정/부정/중립)과 핵심 이벤트 타입을 JSON으로 추출"
- **비용 관리**: 중요 공시(자사주·실적·M&A)만 호출
- 프롬프트 버전 관리 필수 (재현성)

### 종목 매칭 전략

뉴스-종목 매칭은 생각보다 까다롭다.

- **동음이의**: "대림"은 대림산업? 대림B&Co? 대림통상?
- **계열사 영향**: "삼성전자 호실적" → 삼성SDI에도 영향?
- **업종 영향**: "반도체 경기 회복" → 전체 반도체주
- **해외 뉴스**: "엔비디아 어닝" → 국내 AI 관련주

**단계적 해결**:
1. 정확 매칭 (종목명·티커 직접 언급)
2. 별칭 사전
3. 문단 내 여러 종목 언급 시 횟수 가중
4. 업종·테마 매핑은 별도 테이블

### 공시 처리

- DART 공시 유형 코드로 필터링 (전체는 노이즈 과다)
- 주요 유형:
  - `A001` 정기공시 (사업/분기/반기)
  - `A003` 발행공시 (증자·전환사채)
  - `A008` 지분공시 (대량보유)
  - `A004` 주요사항보고
- 본문은 XML/HTML → `dart-fss`, `OpenDartReader`로 파싱
- 실적 공시에서 **영업이익·매출** 자동 추출 → 컨센서스 비교 (가능 시)

### 중복 제거

- 동일 뉴스가 여러 매체에 게재됨
- 기본: 제목 해시
- 고급: SBERT 임베딩 유사도 > 0.85이면 중복 처리

## 2부. 기업평가 (Fundamental Analysis)

### 정량 평가

#### 재무 비율 (DART 재무제표에서 산출)

| 범주 | 지표 |
|---|---|
| 수익성 | ROE, ROA, 영업이익률, 순이익률 |
| 성장성 | 매출·영업이익·순이익 YoY/QoQ |
| 안정성 | 부채비율, 유동비율, 이자보상배율 |
| 활동성 | 총자산회전율, 재고회전율 |
| 가치 | PER, PBR, PSR, EV/EBITDA |

#### 밸류에이션 모델

- **상대가치평가**: 업종 내 멀티플 비교 (MVP 추천)
- **DCF**: 현금흐름 할인. **주의** — 가정 민감도 극단적, 한국 기업은 성장 전제 어려움
- **배당할인모형(DDM)**: 배당주 평가에 한정 사용

#### 팩터 스코어링 (퀀트와 연결)

- 각 지표별 업종 내 분위 계산
- 카테고리별 스코어 → 종합 스코어
- 순위 변화 추적 (레이팅 업/다운)

### 정성 평가

- **공시 변화**: 경영진 변경, 최대주주 변경, 감사인 교체 = 신호
- **감사의견**: 적정·한정·부적정·의견거절 — 부적정/거절은 즉시 제외 유니버스
- **산업 분류**: KRX 업종 + 테마(2차전지·AI 등) — 외부 소스 필요 시 TBD
- **지배구조**: 배당 성향, 자사주 정책, 스튜어드십 코드 적용

### 데이터 품질 이슈

- **리스테이트먼트**: 수정 재무제표 — `as-of` 관리. MVP는 최신값, 백테스트는 당시 시점 스냅샷 필요
- **연결 vs 별도**: 대부분 연결 사용. 별도만 공시하는 종목 예외 처리
- **결산월 차이**: 12월 결산이 대부분이나 일부 상이
- **IFRS 도입 전후**: 2011 이전 데이터 해석 주의

### 평가 결과의 활용

- **매수 후보 선정**: 스크리닝 (예: ROE > 10%, PER < 15, 부채비율 < 100%)
- **비중 조절**: 스코어 상위에 높은 비중
- **워닝 리스트**: 스코어 급락 종목 모니터링

### 무엇을 하지 말 것

- **모델이 예쁘다고 신뢰 금지**. 한국 기업은 회계 투명성 편차 큼
- **과도한 변수**: 팩터 20개보다 핵심 5개가 나은 경우 많음
- **미래 예측**: 컨센서스 추정치는 외부 데이터 의존. MVP 제외

## 엔진 소비자 관점

Strategy Engine은 분석 엔진의 출력을 **피처**로 소비한다:

```python
# 예시
features = {
    "005930": {
        "news_sentiment_1d": 0.32,
        "filing_recent": "자사주매입",
        "roe_ttm": 12.4,
        "per": 11.2,
        "factor_score": 0.78,
    },
    ...
}
```

→ 전략은 이 피처를 입력으로 받아 신호 생성.
→ 피처 계산은 **배치(장 시작 전 1회)** + **이벤트 기반(공시 수신 시)** 혼합.
