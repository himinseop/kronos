# 08. 백테스팅

## 목적

백테스팅은 **"이 전략이 과거에 돈을 벌었을까"**를 답하는 것처럼 보이지만, 실제로는 **"이 전략을 실전에 올렸을 때 얼마나 나빠질 수 있는가"**를 추정하는 도구다. 백테스트가 잘 나왔다는 건 필요조건일 뿐 충분조건이 아니다.

## 프레임워크 후보

| 프레임워크 | 방식 | 장점 | 단점 |
|---|---|---|---|
| **vectorbt** | 벡터화 (pandas/numpy) | 매우 빠름, 파라미터 스윕 용이 | 이벤트 기반 로직 불편 |
| **backtrader** | 이벤트 기반 | 직관적, 복잡한 로직 표현 | 느림, 최근 유지보수 저조 |
| **zipline-reloaded** | 이벤트 기반 | Quantopian 문법, 풍부 | 한국 데이터 어댑터 부재 |
| **자체 구축** | 가변 | 완전한 제어 | 개발 비용 |

## 선택

- **1순위: vectorbt** — 단기·퀀트 전략의 빠른 탐색에 적합
- **보완: backtrader** — 이벤트 기반 전략(뉴스 이벤트 등) 필요 시
- **장기적**: 자체 프레임워크로 통합 (엔진 코드를 백테스트와 실전에서 공유)

### 공통 엔진의 이상향

실전 엔진과 백테스트 엔진이 **같은 Strategy 구현체**를 쓰도록 설계하면, 백테스트가 실전과 유사해진다.

```
Strategy.generate_signals(data) → Signal 리스트
```

- 실전: `data`는 라이브 시세, 주문은 Broker로
- 백테스트: `data`는 과거 데이터, 주문은 시뮬레이터로

## 반드시 반영할 현실 비용

| 항목 | 설명 | 한국 시장 기본값 |
|---|---|---|
| 수수료 | 증권사별 상이 | 0.015% 가정 (KIS 기준) |
| 증권거래세 | 매도 시 | **0.18%** (KOSPI/KOSDAQ 2024 기준, 변동 추적 필요) |
| 슬리피지 | 시가와 체결가의 차이 | 유동성에 반비례. 보수적으로 0.1~0.3% |
| 농특세 | KOSPI 매도 시 | 0.15% (거래세에 포함) |
| 체결 지연 | 시그널부터 체결까지 | 1~3초. 분봉 전략은 영향 큼 |
| 호가 단위 | 가격별 다름 | 주문가 조정 로직 필요 |
| 유동성 제약 | 거래대금 10% 이상 주문 시 시장 영향 | 주문 크기 제한 |

**세금 최신화**: 세율은 바뀔 수 있음. 설정 파일로 분리해 쉽게 업데이트.

## 흔한 편향(Pitfall)과 방지

### 1. 룩어헤드 편향 (Look-ahead Bias)
- **증상**: 미래 데이터를 피처로 사용. 예: 2024 Q1 전략에 2024 Q2 재무 사용
- **방지**: 모든 데이터에 `as_of_date` 기록, 신호 생성 시 `t` 시점까지만 사용
- **재무 데이터**: 공시 시점 이후만 유효 (분기 종료 후 45일 이내 공시가 일반)

### 2. 생존 편향 (Survivorship Bias)
- **증상**: 현재 상장 종목만으로 백테스트 → 파산/상장폐지 종목 누락
- **방지**: 과거 상장 종목 포함한 역사적 유니버스 구축 (KRX 상장폐지 목록)

### 3. 과적합 (Overfitting)
- **증상**: 파라미터 튜닝으로 과거 수익률 극대화 → 실전에서 붕괴
- **방지**:
  - **훈련/검증/테스트 분할**
  - **워크포워드 분석**: 롤링 윈도우로 훈련·검증 반복
  - 파라미터 개수 최소화
  - 백테스트 기간은 여러 시장 국면(상승/하락/횡보) 포함

### 4. 선택편향
- **증상**: 잘 되는 종목만 골라서 백테스트
- **방지**: 유니버스는 **규칙 기반**으로 선택 (예: "시총 상위 200개")

### 5. 데이터 스누핑
- **증상**: 같은 데이터로 100가지 전략 테스트 → 하나는 우연히 좋게 나옴
- **방지**: 가설 수 × 유의수준 보정 (Bonferroni 등), 또는 아웃오브샘플 기간 엄격 분리

### 6. 비현실적 체결 가정
- **증상**: 종가 체결 가정 → 실전은 슬리피지·지연
- **방지**: **다음 봉 시가 체결** 같은 보수적 규칙 + 슬리피지 모델

## 평가 지표

- **누적 수익률** (CAGR)
- **샤프 비율** (연환산)
- **소티노 비율** (하방 변동성 기준)
- **최대낙폭(MDD)** 및 회복 기간
- **승률 · 평균 손익비**
- **전환율(turnover)** — 거래비용 민감도
- **베타 · 알파** — 시장 대비
- **롤링 성과** — 특정 구간 의존도 확인

## 워크포워드 분석 예시

```
[훈련 1년] [검증 3개월] → 파라미터 확정 → [다음 3개월 실측]
           ↓ 윈도우 이동
[훈련 1년] [검증 3개월] → ...
```

- 실측 구간만 성과 집계 → 현실에 가까운 추정
- 파라미터가 구간마다 크게 바뀐다면 **불안정한 전략**

## 리포트 아웃풋

- 주식 대비 벤치마크(KOSPI200) 수익률 차트
- 드로다운 차트
- 월별/연별 수익률 히트맵
- 거래 상세 테이블 (진입·청산·보유기간·수익률)
- 전략별 성과 분해

## 제약

- 과거에 없던 이벤트는 백테스트 불가 (블랙스완)
- 뉴스·공시 기반 전략은 **과거 데이터 확보 난이도**가 큼
- 유동성 낮은 종목은 실제 체결 불가능할 수 있음 → 유동성 필터 전제

## 백테스트와 실전의 격차 관리

- 모의투자 결과와 백테스트 결과를 **같은 기간**에 비교
- 두 결과가 크게 다르면 **슬리피지·지연·체결 모델** 재조정
- 이 피드백 루프 없이는 백테스트는 숫자 놀이로 전락
