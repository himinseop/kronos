# 14. 로드맵

## 철학

- 수익보다 **시장 이해와 운영 안정성**을 먼저
- **뉴스·공시 분석으로 시장의 흐름을 먼저 읽은 뒤** 매매에 진입
- 각 Phase는 명확한 **Definition of Done(DoD)**을 가지고, 충족 전에는 다음으로 넘어가지 않음
- 모의투자 검증 없는 실전 전환은 절대 금지

## Phase 0 — 환경 설정 (약 1주)

**목표**: 뉴스·공시 수집에 필요한 최소 환경 구축. **KIS 계정은 이 단계에서 제외** — Phase 4(시세·매매 진입) 직전에 별도로 진행 ([05-broker-integration.md](../2-design/05-broker-integration.md)).

### 작업
- [ ] DART Open API 키 발급 (https://opendart.fss.or.kr)
- [ ] 네이버 검색 API 키 발급 (https://developers.naver.com)
- [ ] 파이썬 프로젝트 스캐폴딩 (`uv init`, 디렉토리, pyproject, pre-commit)
- [ ] `.env.example`, `.gitignore`, 로깅·설정 로딩 기본 구조
- [ ] CLI 스켈레톤 (`kronos status`)

### Definition of Done
- `uv run kronos status`가 DART / 네이버 API **연결 성공** 출력
- pre-commit이 동작하며 비밀 탐지 통과
- 저장소 public이 되어도 문제 없는 상태 (비밀 없음)

> **메모**: KIS 종목뉴스 API는 보유 계좌가 있으면 유용하지만 Phase 1 MVP에는 **불필요**. DART + 네이버 + RSS만으로 시장 흐름은 충분히 관찰 가능.

## Phase 1 — 뉴스·공시 수집 파이프라인 + 수집 대시보드 (1~2주)

**목표**: 한국 시장 뉴스·공시를 매일 자동으로 수집하고 종목별로 매핑하여 저장. **수집 즉시 대시보드에서 모니터링**.

### 작업

#### 1-A. 수집기
- [ ] DART 공시 수집기 (당일 공시 폴링, 30초 주기)
- [ ] 네이버 뉴스 수집기 (관심 종목 기준 키워드 검색)
- [ ] 언론사 RSS 1~2개 통합 (한경/매경 등)
- [ ] **SQLite 저장** + 해시(SHA-256 of 정규화 제목+URL) 기반 중복 제거
- [ ] 종목 매칭 사전 (KRX 전체 종목 + 별칭 맵)
- [ ] APScheduler로 장중·장후 주기적 수집 등록
- [ ] 구조화 로그 + 수집 실패 알림

> KIS 종목뉴스 API는 Phase 4(KIS 통합) 이후에 같은 인터페이스로 추가. Phase 1에서는 의도적으로 제외.

#### 1-B. 수집 모니터링 대시보드 (Streamlit, 로컬)
- [ ] **개요 탭**: 소스별(DART/네이버/RSS) 시간대별 수집 건수, 최근 24h 추세, 오늘 수집 합계
- [ ] **소스 헬스**: 각 수집기의 마지막 성공 시각, 연속 실패 수, API 호출 한도 사용률
- [ ] **최근 뉴스·공시 피드**: 시간 역순 테이블 (시각 / 종목 / 제목 / 출처 / 원문 링크), 종목·소스·기간·키워드 필터
- [ ] **공시 유형 분포**: DART `pblntf_ty` 코드별 건수 (실적/증자/지분 등)
- [ ] **중복·매칭 품질**: 해시 중복률, 종목 매칭 성공률(매칭됨/미매칭), 미매칭 샘플 목록
- [ ] **원본 뷰어**: 공시·뉴스 1건 클릭 시 본문·메타 표시 (디버깅용)

### Definition of Done
- 1주 이상 무인 운영, 일평균 수집량 안정적
- 중복률 5% 이하 (해시 검증)
- 종목 매칭 정확도 > 90% (샘플 100건 수동 검증)
- **대시보드 한 화면에서 "오늘 수집이 정상인가"가 즉시 판단됨** (수동 DB 쿼리 불필요)

## Phase 2 — 감성분석 + 분류 + 대시보드 확장 (1~2주)

**목표**: 수집된 뉴스·공시를 한국어 금융 특화 모델로 감성 점수화하고 카테고리 분류. 대시보드에 감성·카테고리 뷰 추가.

> **진행 메모 (B경로 채택)**: PostgreSQL 전환은 실제 동시쓰기 병목이 생길 때로 **연기**하고,
> SQLite를 유지한 채 감성분석부터 진행. LLM 분류는 외부 API 대신 **자체(로컬) 모델** 사용.

### 작업

#### 2-A. 분석 파이프라인
- [x] **KR-FinBERT** 적용 (`snunlp/KR-FinBert-SC`) — `analysis/sentiment.py`
- [x] `sentiments` 테이블 + 감성 점수 배치 워커 (`analysis/run.py`)
- [x] sentiment 컨테이너 (torch, CPU) — compose `sentiment` 서비스, 백로그 자동 소진 루프
- [x] 카테고리 분류 — **자체 LLM** (Ollama + Qwen2.5-3B, OpenAI 호환 /v1) —
  `analysis/classify.py` + `classify_run.py`. 종목 매칭 뉴스만 최신순 배치 처리
  (chunk 15, rationale 생략, ~1.1s/건). sentiments 테이블에 `model='cat:<모델>'`로 적재.
  Ollama는 mycomai 등과 공유 가능한 호스트 네이티브 서비스(GPU/Metal 가속).
- [ ] 모델 라벨링 샘플(100건) 수동 평가 — 카테고리·감성 정확도
- [ ] (연기 해제·완료) SQLite → PostgreSQL 마이그레이션 (손상사고로 조기 완료)

#### 2-B. 대시보드 확장 (Phase 1 대시보드 위에 추가)
- [x] **감성 탭**: 커버리지·라벨 분포·일별 추세·종목별 상하위·점수 피드
- [x] **카테고리 탭**: 커버리지·분포·일별 스택 추세·카테고리별 상위 종목·분류 피드
- [ ] **모델 비교 뷰**: 룰 이벤트 vs 감성 vs LLM 카테고리 교차 검토

### Definition of Done
- [x] 전 수집 뉴스에 감성 점수 부여 (백로그 소진 중)
- [ ] 카테고리 분류 정확도 검증 (수동 평가 기준)
- [x] 대시보드에서 감성 기준 필터·검색 가능

## Phase 3 — 시장 흐름 분석 뷰 + 알림 (1주)

**목표**: Phase 1~2 대시보드 위에 **시장 흐름을 읽는 분석 뷰**를 얹고, 중요 이벤트 알림 자동화. 자동매매 진입 전 **수동 매매 보조 도구**로 1~2개월 사용.

### 작업
- [ ] 일별·종목별·섹터별 감성 집계 SQL 뷰 (Materialized View)
- [ ] 대시보드 **시장 흐름 탭** 추가:
  - [ ] 시장 전체 감성 히트맵 (섹터×일자)
  - [ ] 종목 상세 페이지 (감성 추세 차트, 최근 뉴스·공시 타임라인, 관련 재무 요약)
  - [ ] Top Movers: 오늘 감성 z-score 급등·급락 종목 랭킹
  - [ ] 중요 공시 피드 (실적/M&A/자사주/최대주주 변경 필터링)
- [ ] 급변 알림 (감성 z-score 급등락, 중요 공시 발생 → 텔레그램)
- [ ] 일일 시장 요약 리포트 (장 마감 후 자동 생성)

### Definition of Done
- 매일 장 마감 후 리포트 자동 발송
- 사용자가 대시보드만 보고 시장 흐름 파악 가능
- **이 단계에서 1~2개월 운영하며 데이터 품질·시그널 유효성 체감 후 Phase 4로 진행**

## Phase 4 — KIS 통합 + 시세 수집 + 백테스트 인프라 (2~3주)

**목표**: 매매 전략 개발에 필요한 시세 데이터와 백테스트 엔진 준비. **이 시점에 KIS 계정과 앱키를 발급**.

### 4-A. KIS 계정·인증 (선행)
- [ ] 한국투자증권 신규 계좌 개설 (비대면)
- [ ] KIS Developers 가입, **모의투자 앱키/시크릿** 발급
- [ ] 모의투자 계좌 신청
- [ ] 접근토큰 발급/갱신 루틴
- [ ] `kronos status`에 KIS 연결 점검 추가
- [ ] KIS 종목뉴스 API 수집기 추가 (Phase 1 인터페이스에 어댑터로 연결)

### 4-B. 시세 + 백테스트
- [ ] 과거 일봉·재무 적재 (`pykrx`, `FinanceDataReader`, DART)
- [ ] KIS WebSocket 실시간 시세 구독 (관심 종목 한정)
- [ ] 생존편향 제거 유니버스 (상장폐지 종목 포함)
- [ ] `vectorbt` 기반 백테스트 엔진
- [ ] 슬리피지·수수료·세금 모델
- [ ] 워크포워드 분석 유틸

### Definition of Done
- KIS 모의투자 인증·시세 조회 성공
- KIS 종목뉴스가 기존 수집 대시보드의 새 소스로 표시됨
- 과거 5년치 일봉 데이터 적재 완료
- 단순 전략(이동평균 크로스) 백테스트 1회전 완료, 리포트 생성

## Phase 5 — 전략 개발 (3~4주)

**목표**: Phase 1~3에서 축적한 뉴스·감성 데이터를 활용한 전략 1~2개 백테스트 통과.

### 우선 전략
1. **뉴스 트리거 + 기술지표 결합**: 긍정 뉴스 급증 + RSI 과매도 → 매수
2. **공시 이벤트 반응**: 자사주 매입·실적 서프라이즈 후 단기 모멘텀
3. **펀더멘털 + 감성**: 저PER·고ROE 종목 중 감성 상위

### Definition of Done (전략별)
- 백테스트에서 벤치마크(KOSPI200) 대비 샤프비율 개선
- 슬리피지·수수료·세금 반영
- 워크포워드 검증 통과

## Phase 8 — 모의투자 무인 운영 검증 (4주)

**목표**: 실전 진입 전 모의투자에서 1개월 무인 운영 통과.

### 작업
- [ ] 매일 장 마감 후 일일 요약 리포트 자동 발송
- [ ] 주간 성과 리뷰 (전략별 PnL, 리스크 이벤트, 시스템 장애)
- [ ] 백테스트 결과 vs 모의투자 결과 비교 (편차 추적)

### Definition of Done
- **4주 연속 무인 운영, 치명적 장애 0건**
- 모의 vs 백테스트 수익률 편차 0.5%p 이내
- 리스크 엔진 차단 사례 모두 적절했음을 사후 확인

## Phase 9 — 소액 실전 전환 (지속)

**목표**: 점진적 자본 확장. 실전은 **모든 검증 통과 후** 시작.

### 단계
1. **1주차**: 실전 자본의 **5%만**, 1개 전략만
2. **2~4주차**: 문제 없으면 자본 20%, 2개 전략
3. **2~3개월차**: 50%까지 확장
4. **그 이후**: 백테스트/모의/실전 편차 모니터링하며 조정

### 실전 투입 체크리스트
- [ ] 모의투자 4주 무인 운영 통과
- [ ] 모든 리스크 차단 시나리오 검증됨
- [ ] **실전 API 키가 모의 키와 분리**되어 있음 재확인
- [ ] 일일 손실 한도 보수적으로 설정 (자본의 -2% 등)
- [ ] 텔레그램 알림 정상 동작
- [ ] 장애 대응 플레이북 숙지 ([11-operations.md](../4-operations/11-operations.md))

## Phase 10 — 미국 시장 확장 (시점 미정)

**목표**: 한국 시장 운영이 안정된 후 미국 시장으로 확장.

### 검토 항목
- **데이터 소스**: SEC EDGAR(공시), Finnhub/Benzinga(뉴스), Polygon/Alpaca(시세)
- **증권사**: KIS 해외주식 vs IBKR vs Alpaca 비교
- **시간대**: 미국장 야간 운영 → 스케줄러·알림 시각 조정
- **세금**: 미국 주식 양도소득세, 환율 영향
- **언어**: 영문 뉴스 → FinBERT(영문) 또는 LLM 직접 분석

### 진입 조건
- 한국 시장 Phase 9가 **3개월 이상 안정 운영**
- 한국에서 검증된 전략·인프라 패턴을 미국에 이식 가능한 형태로 추상화 완료

## 일정 가이드라인

**보수적 추정** (주말·저녁 기준):
- Phase 0~3 (뉴스·공시 + 감성 + 모니터링): **약 1.5~2개월**
- Phase 3 운영 관찰: **추가 1~2개월** (이 기간 동안 시장 흐름 체득)
- Phase 4~7 (백테스트 + 주문 + 리스크): **약 2~3개월**
- Phase 8 (모의투자 검증): 4주
- Phase 9 (실전): 지속

**총 실전 진입까지 약 4~6개월**.
일정 지연은 나쁘지 않지만, **DoD를 건너뛰는 것은 금지**.

## 핵심 원칙
