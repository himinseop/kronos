## 결정

- **손상 대응은 PG 전환(B안)**: SQLite 유지+락 회피보다 근본 해결 선택. macOS Docker
  bind mount 위 SQLite 다중 프로세스 접근은 금지(VirtioFS가 WAL 락 미보장).
- **Ollama는 Docker 아닌 네이티브**: macOS Docker는 M1 GPU(Metal) 미지원 → CPU only로
  5~10배 느림. GPU 가속 위해 네이티브 필수. Linux+NVIDIA 이전 시 컨테이너화 재검토.
- **분류 저장은 sentiments 테이블 재사용**: 스키마에 이미 `category`/`rationale` 컬럼 존재,
  `model` 값으로 감성/카테고리 구분(UNIQUE(target_type,target_id,model)).
- **속도 최적화로 rationale 생략**: 대량 백필(112K)에서 근거 문장 생성이 출력 토큰의
  대부분 → 생략 시 3.4s→1.1s/건. 근거는 옵션으로 유지.
- **mycomai 연동은 미관여**: 동일 Ollama 서버 공유 가능하나, 연동 작업은 mycomai
  프로젝트에서 별도 진행하기로.

## 이슈

- **Docker VM 레지스트리 네트워크 고장(미해결)**: `docker pull`/`build`가 metadata
  단계에서 hang. 호스트 curl·컨테이너 일반 outbound는 정상. Docker Desktop 재시작으로도
  미해소. → 스택을 컨테이너 대신 **호스트 프로세스**로 운영 중(postgres만 컨테이너).
- **분류 품질 ~75%(3B, 헤드라인 기준)**: "상생/협력사" 류가 product로 몰리는 경향.
  대량 1차 분류엔 충분. 정확도 부족 시 7B 승격 또는 카테고리 설명 튜닝.
