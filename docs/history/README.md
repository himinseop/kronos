# 작업 히스토리

일자별 작업 기록을 남기는 디렉토리. 무엇을·왜 했는지의 시간순 로그.

## 규칙

- 파일명: `YYYY-MM-DD.md` (작업한 날짜). 하루에 여러 세션이면 한 파일에 이어 적는다.
- 각 파일 상단에 그날의 **한 줄 요약**, 이어서 **한 일 / 결정 / 이슈 / 다음** 순으로.
- 커밋 해시를 함께 남겨 코드 변경과 연결한다.
- 이 로그는 "왜 이렇게 했나"를 위한 것. 현재 시스템 상태는 [`../STATUS.md`](../STATUS.md),
  전체 계획은 [`../14-roadmap.md`](../5-roadmap/14-roadmap.md)를 본다 (중복 서술 최소화).
